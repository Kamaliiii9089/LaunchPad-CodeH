# Collaboration Features - Quick Start Guide

## 🚀 Get Started in 5 Minutes

### 1. View Security Events
Navigate to the **Threats** tab in your dashboard to see all active security events.

### 2. Add Your First Comment
1. Find any security event
2. Click the **💬 Comment** button
3. Type your message
4. Use `@username` to notify team members
5. Click **Post Comment**

**Try it now**: Add a comment like "Investigating this issue. @john please review the logs."

### 3. Start a Shared Investigation
1. Click **🔍 Investigate** on a security event
2. Enter investigation details:
   - **Title**: "Investigation: <Threat Type>"
   - **Description**: Brief scope and objectives
   - **Priority**: Choose appropriate level
3. Click **Create Investigation**

**Pro Tip**: Assign team members immediately for faster response.

### 4. Add Findings to Investigation
1. Open any investigation
2. Click **+ Add Finding**
3. Document what you've discovered
4. Set severity level
5. Click **Add Finding**

**Example Finding**:
- Title: "Attack source identified"
- Description: "Traffic originating from IP 192.168.1.100"
- Severity: High

### 5. Access Knowledge Base
1. Click **📚 KB** on any event (or from Help tab)
2. Browse articles or search for topics
3. Follow step-by-step procedures
4. Click **👍 Yes, helpful** to provide feedback

**Quick Access**: The KB automatically shows articles related to the event type you're viewing.

## 💡 Quick Tips

### Effective Communication
- **Use @mentions wisely**: Only notify people who need to act
- **Mark sensitive info as internal**: Check "Internal note (team only)" for private discussions
- **Be specific**: Include relevant details, IPs, timestamps, etc.

### Investigation Best Practices
- **Update status regularly**: Keep team informed of progress
- **Document as you go**: Don't wait until the end to add findings
- **Set correct priority**: Helps team focus on what matters most
- **Use timeline**: Review what happened and when

### Knowledge Base Usage
- **Search first**: Before asking, check if there's documentation
- **Rate articles**: Help improve content quality
- **Reference in comments**: Share helpful KB articles with team

## 🎯 Common Workflows

### Quick Comment & Notify
```
1. Click 💬 Comment
2. Type: "@sarah can you check if this is a false positive?"
3. Post
```

### Full Investigation
```
1. Click 🔍 Investigate
2. Fill details + assign team
3. Reference KB article
4. Add findings as discovered
5. Update status to Resolved
```

### Find & Follow Procedure
```
1. Click 📚 KB
2. Search for threat type (e.g., "DDoS")
3. Follow numbered steps
4. Document actions in investigation
```
